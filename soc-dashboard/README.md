# LLM-SOC Dashboard: Hybrid SOC Alert Triage Web Demo

This folder contains the web-based dashboard for the **LLM-SOC Hybrid Alert Triage PoC**.

The dashboard demonstrates how a SOC analyst can submit a security alert, receive an incident-grade prediction from a trained machine-learning model, and view analyst-facing explanation and response guidance generated by Qwen through Ollama.

---

## Dashboard Overview

The dashboard uses a hybrid architecture:

```text
Browser Frontend
HTML / CSS / JavaScript
        ↓
FastAPI Backend
        ↓
TF-IDF + Logistic Regression Classifier
        ↓
Qwen via Ollama
        ↓
Hybrid SOC Triage Output
````

The system separates classification and explanation:

| Component                    | Role                                            |
| ---------------------------- | ----------------------------------------------- |
| TF-IDF + Logistic Regression | Predicts the incident grade                     |
| Qwen via Ollama              | Generates analyst summary and response guidance |
| FastAPI                      | Serves backend API and frontend dashboard       |
| JavaScript frontend          | Provides user interface for SOC triage demo     |

---

## Main Features

The dashboard supports:

| Feature              | Description                                  |
| -------------------- | -------------------------------------------- |
| Load sample alerts   | Load examples from `llm_eval_subset.csv`     |
| Paste custom alert   | Manually enter an `alert_text` record        |
| Run hybrid triage    | Send alert to FastAPI backend                |
| ML prediction        | Show predicted incident grade                |
| LLM analyst summary  | Generate plain-English explanation           |
| Risk explanation     | Explain why the alert may matter             |
| MITRE interpretation | Provide possible MITRE ATT&CK interpretation |
| Recommended actions  | Suggest response steps                       |
| Containment priority | Show Low, Medium, or High priority           |
| Raw JSON output      | Display raw LLM response for auditability    |

## Ollama Setup

The dashboard uses Qwen through Ollama for explanation generation.

Check installed Ollama models:

```bash
ollama list
```

Recommended model:

```text
qwen2.5:7b-instruct
```

Start Ollama:

```bash
ollama serve
```

Test the model:

```bash
ollama run qwen2.5:7b-instruct
```

## Backend API Endpoints

| Endpoint           | Method | Description                         |
| ------------------ | ------ | ----------------------------------- |
| `/`                | GET    | Loads the dashboard frontend        |
| `/api`             | GET    | Returns backend API information     |
| `/health`          | GET    | Checks model and sample data status |
| `/samples`         | GET    | Loads sample alerts                 |
| `/samples/{index}` | GET    | Loads one sample alert              |
| `/predict`         | POST   | Runs ML prediction only             |
| `/triage`          | POST   | Runs full hybrid ML + LLM triage    |
| `/docs`            | GET    | Opens FastAPI Swagger documentation |

## Dashboard Workflow

The dashboard workflow is:

```text
1. User opens localhost:8000
2. User loads a sample alert or pastes a custom alert
3. User clicks Run Hybrid Triage
4. FastAPI receives the alert text
5. Logistic Regression predicts the incident grade
6. Qwen generates analyst explanation and response guidance
7. Backend returns structured JSON
8. Dashboard displays final SOC triage result
```

## Frontend Configuration

The frontend is served directly by FastAPI. Therefore, the API base URL in `frontend/app.js` should be:

```javascript
const API_BASE_URL = "";
```

This allows the frontend to call backend endpoints directly:

```javascript
/samples
/triage
/health
```

---

## Backend Configuration

Main configuration values are defined in `backend/app.py`:

```python
MODEL_PATH = BASE_DIR / "models" / "tfidf_logreg_baseline.joblib"
SAMPLE_PATH = BASE_DIR / "data" / "processed" / "llm_eval_subset.csv"
FRONTEND_DIR = BASE_DIR / "frontend"

OLLAMA_URL = "http://localhost:11434/api/generate"
OLLAMA_MODEL = "qwen2.5:7b-instruct"

HOST = "0.0.0.0"
PORT = 8000
```

Change `OLLAMA_MODEL` if you want to use another local model:

```python
OLLAMA_MODEL = "qwen2.5:3b-instruct"
```

or:

```python
OLLAMA_MODEL = "qwen3:4b"
```

## Troubleshooting

### Ollama connection error

Make sure Ollama is running:

```bash
ollama serve
```

Then test:

```bash
ollama run qwen2.5:7b-instruct
```

---

### Model file not found

Check that this file exists:

```text
models/tfidf_logreg_baseline.joblib
```

If it does not exist, train the baseline model:

```bash
python src/03_train_baseline.py
```

---

### Sample data not found

Check that this file exists:

```text
data/processed/llm_eval_subset.csv
```

If it does not exist, create it:

```bash
python src/04_create_llm_eval_subset.py
```

---

### Frontend not loading

Check that the frontend folder exists:

```text
frontend/
├── index.html
├── app.js
└── style.css
```

Then restart the backend:

```bash
python backend/app.py
```

---

### Port already in use

If port `8000` is already used, stop the existing process or change the port in `backend/app.py`:

```python
PORT = 8001
```

Then open:

```text
http://localhost:8001
```

## Notes

The dashboard is a proof-of-concept system. It does not connect to a live SIEM, EDR, firewall, or cloud monitoring system.

The LLM-generated recommendations are intended as analyst guidance only. They should not be treated as automatic remediation actions.

The raw JSON output is shown to support transparency, debugging, and auditability.
